Q1. 如何有 NtxDisplaySettings 的？

入口：SettingsHomepageActivity.java

當使用者在左側 Tab 點擊 「Display」時，SettingsHomepageActivity 會比對 title 字串，然後 changeContent() 替換右側 Fragment：

// SettingsHomepageActivity.java (Line 279-280)
} else if (title.equals(getString(R.string.display_settings))) {
    changeContent(new NtxDisplaySettings(), bundle);
}

Fragment 本體：NtxDisplaySettings.java

它繼承自 DashboardFragment，指定載入 R.xml.ntx_display_settings 並建立 Controller 列表：

// NtxDisplaySettings.java
@SearchIndexable
public class NtxDisplaySettings extends DashboardFragment {

    @Override
    protected int getPreferenceScreenResId() {
        return R.xml.ntx_display_settings;  // ← 指定 XML
    }

    @Override
    protected List<AbstractPreferenceController> createPreferenceControllers(Context context) {
        return buildPreferenceControllers(context, getSettingsLifecycle());
    }
}

Q2. 裡面的 Preference 有哪些？

ntx_display_settings.xml

# Preference 類型 key Controller 說明
1 FrontLightPreference front_light —（自行管理） 前光亮度 + 色溫雙 SeekBar
2 NtxColorPreference ntx_color_settings —（自行管理） E-ink 色彩 / 對比度調整
3 ToolsPickActivityPreference — — 自定義快捷鍵設定
4 SwitchPreference hall_sensor_switch ProtectiveCaseWakeUpPreferenceController 保護殼喚醒開關
5 NtxRestrictedListPreference screen_timeout SleepTimeoutController 螢幕逾時（單選 Dialog）
6 NtxRestrictedListPreference sleep_screen_mode SleepScreenController 休眠畫面（單選 Dialog）
7 NtxCfaModeRestrictedListPreference ntx_color_mode NtxColorModePreferenceController CFA 色彩模式（單選 Dialog）
💡 規則：凡是會彈出 Dialog 的 ListPreference，一律使用 NtxRestrictedListPreference，確保 E-ink 樣式（72dp 行高 / 黑色圓圈 Radio）。

Q3. 如何引入 Divider 的？

位置：SettingsPreferenceFragment.java

所有設定頁的 Fragment 都繼承自 SettingsPreferenceFragment，我們在這裡統一注入分隔線。

Step 1. 停用系統預設 Divider

// SettingsPreferenceFragment.java → onViewCreated()
SharedPreferences tempPref = getContext().getSharedPreferences(PREF_NTXSETTINGS, Context.MODE_PRIVATE);
if (tempPref.getBoolean(DEFAULT_SETTINGS_ITEMS, true)) {
    setDivider(null);       // 拿掉系統預設的灰色 divider
    setDividerHeight(0);   // 高度歸零
}

Step 2. 自訂 ItemDecoration

// SettingsPreferenceFragment.java → onCreateRecyclerView()
if (tempPref.getBoolean(DEFAULT_SETTINGS_ITEMS, true)) {
    recyclerView.addItemDecoration(new RecyclerView.ItemDecoration() {
        private Paint paint = new Paint();
        { paint.setColor(Color.BLACK); }

        @Override
        public void getItemOffsets(Rect outRect, View view, ...) {
            outRect.top    = 20;   // 上方間距 20px
            outRect.bottom = 20;   // 下方間距 20px
            outRect.left   = 32;   // 左邊距 32px
            outRect.right  = 32;   // 右邊距 32px
        }

        @Override
        public void onDraw(Canvas c, RecyclerView parent, ...) {
            for (int i = 0; i < parent.getChildCount(); i++) {
                View child = parent.getChildAt(i);
                // 項目底部畫一條 2px 寬的黑線
                c.drawRect(
                    32,                        // left
                    child.getBottom() + 18,    // top
                    parent.getWidth() - 32,    // right
                    child.getBottom() + 20,    // bottom (線寬 = 20-18 = 2px)
                    paint
                );
            }
        }
    });
}
📌 條件控制：只有 SharedPreferences PREF_NTXSETTINGS 中 DEFAULT_SETTINGS_ITEMS == true 時才生效（也就是 Ntx 模式），一般 AOSP 模式不會引入。

Q4. 如何 Remove ActionBar？

Theme 繼承鏈

ActionBar 的移除透過 Theme 繼承完成，不需要在 Java 層做任何事情：

R.style.Theme_NtxSettings); Theme.NtxSettings parent: Theme.Settings.NoActionBar （定義在 styles.xml） windowActionBar = false · windowNoTitle = true

themes.xml 中的定義

<!-- themes.xml -->
<style name="Theme.Settings.NoActionBar">
    <item name="android:windowActionBar">false</item>
    <item name="android:windowNoTitle">true</item>
</style>

styles.xml 中的 NtxSettings

<!-- styles.xml -->
<style name="Theme.NtxSettings" parent="Theme.Settings.NoActionBar">
    <item name="android:textAppearanceListItem">@style/TextAppearance.NtxListItem</item>
    <item name="android:textAppearanceListItemSecondary">@style/TextAppearance.NtxListItemSecondary</item>
    <item name="alertDialogTheme">@style/Theme.NtxAlertDialog</item>
    <item name="materialAlertDialogTheme">@style/Theme.NtxAlertDialog</item>
</style>
📌 關鍵：setTheme() 必須在 super.onCreate() 之前呼叫，否則 Window 的 ActionBar 已經建立完成就來不及了。

Q5. 如何設定 Dialog Style？

整體繼承鏈

Adapter 注入：NtxRestrictedListPreference.java

光靠 Theme 只能控制 Dialog 的外觀框架，列表項目內部的 Layout 需要透過覆寫 createListAdapter() 來注入：

public class NtxRestrictedListPreference extends RestrictedListPreference {

    @Override
    protected ListAdapter createListAdapter(Context context) {
        return new NtxRestrictedArrayAdapter(context, getEntries(), getSelectedValuePos());
    }

    public class NtxRestrictedArrayAdapter extends ArrayAdapter<CharSequence> {
        public NtxRestrictedArrayAdapter(Context context, CharSequence[] objects, int selectedIndex) {
            super(context,
                R.layout.ntx_select_dialog_singlechoice,  // ← 自訂 Layout
                android.R.id.text1,                      // ← 系統 namespace ID
                objects);
        }
    }
}
📌 總結：Dialog Style = Theme 控制外框 + Adapter 控制列表項
• 外框：Theme.NtxAlertDialog → windowBackground / colorBackgroundFloating / elevation
• 標題：NtxDialogTitle style → ntx_dialog_title_bg.xml（底部黑線）
• 列表項：NtxRestrictedListPreference → createListAdapter() → ntx_select_dialog_singlechoice.xml（72dp 行高 + 40dp Radio）
