▎AOSP 原生下拉選單

Quick Settings Tiles (WiFi、BT、飛航…) + BrightnessSlider + 少量按鈕（編輯、設定）。
通知面板與 QS 面板是上下堆疊的單一畫面。

▎NTX 客製後的下拉選單

改為左右分割畫面：左邊 = 通知列表，右邊 = QS (Tiles + 自訂控制面板)。
QS Footer 從原本幾乎空白的底部，變成一個完整的控制中心，內含：

  • 🔊 音量區：開關 + Media / Ring / Alarm 三條 SeekBar
  • 💡 前光區：開關 + 亮度 SeekBar + 色溫 SeekBar
  • 🌗 顯示區：對比度 (Gamma) SeekBar + 白滴 (Whiteness) SeekBar

8 關鍵設計決策
決策原生做法NTX 做法原因
面板佈局 上下堆疊 左右分割 E-ink 螢幕刷新慢，減少滾動操作
亮度控制 BrightnessController
(0.0~1.0 浮點)		 直操 SeekBar
(0~255 整數)		 對齊 E-ink 驅動原生值域
同步機制 BroadcastReceiver ContentObserver 實體按鍵不發廣播，需監聽 DB
QS Tile 風格 Material 3 圓角漸層 黑框白底 E-ink 無法顯示彩色漸層
StatusBar 僅顯示系統圖示 內嵌 6 個按鈕 E-ink 裝置無實體導航列
Recent Apps 原生 Overview 自訂 Activity 原生動畫在 E-ink 上體驗差
