01 · 緣起

原本的做法，為什麼不行？

❌ 舊做法：HTML 放 GitHub Pages

  • 簽到資訊寫在網頁裡
  • 換一台電腦，資料就不見了
  • 別人簽的，自己這台看不到
根因：GitHub Pages 是「靜態網站」，只能讀、不能寫，沒有後端、沒有資料庫——資料其實只存在「那台瀏覽器」裡。

✅ 我們要的

  • 掃 QR 即可線上簽到
  • 所有人的資料集中保存
  • 任何電腦都看到同一份
  • 自動產出缺勤報表

04 · 技術架構

用到哪些技術

🖥️ 前端：純 HTML / CSS / JS
簽到頁、管理台、報表頁，無框架
⚙️ 後端：Flask（Python）

場次 / 名冊 / 簽到三張表
🔳 QR：segno　｜　🌐 對外：cloudflared

為什麼這樣選

  • 輕量：整套能跑在一台機器

  • 可成長：日後要換 MySQL／加功能都容易

06 · 部署方案 A（非 Docker）

本機直接跑：最快上手

在自己電腦啟動後端，再用 cloudflared 開一條公開網址。適合：自己操作、臨時開會。

  1. 執行 ./run.sh
  2. 自動啟動後端 + Cloudflare Tunnel
  3. 取得公開網址 https://xxxx.trycloudflare.com
  4. 投影 QR → 大家掃 → 簽到
# 一行啟動
ATTEND_ADMIN_KEY="你的金鑰" ./run.sh
缺點：電腦要全程開著、網址每次重開會變、需自己設好 Python 環境。
07 · 觀念

什麼是 Docker？為什麼需要它？

把「作業系統 + Python + 所有套件 + 我們的程式」整包封裝成一個 image（映像檔）。

沒有 Docker，給別人時…

  • 對方要自己裝對的 Python
  • pip 安裝套件、對版本
  • 設定環境、踩各種坑
  • 👉「在我電腦明明可以」

有 Docker，對方只要…

# 就這一行
docker compose up -d --build
  • 電腦完全不用裝 Python
  • Mac / Windows / Linux 結果一模一樣
比喻：Docker 像「貨櫃」——不管搬到哪艘船（哪台電腦／哪個雲），裡面的東西原封不動、照常運作。
08 · 部署方案 B（Docker）

Docker 版：給別人用 / 搬上雲

# 啟動（本機或任何伺服器）
docker compose up -d --build

# 停止 / 移除
docker compose stop
docker compose down

內含 Dockerfile、compose、名冊、健康檢查，
以 gunicorn（正式級）執行。

比較本機直跑Docker
對方裝 Python/套件要免
跨電腦一致常出包一致
搬上雲端要重設直接跑
執行品質開發伺服器gunicorn
09 · 對外存取

「外部的人能不能用」取決於跑在哪

Docker 本身不會讓它對外——能不能被連到，看你把容器跑在哪台機器、有沒有對外的路。

容器跑在…誰連得到外部可用穩定度費用
你筆電只有自己 / 同 WiFi否—免費
你筆電 + Tunnel任何人（公開網址）是筆電要開著免費
OCI 雲端 VM任何人，固定網址是24 小時永久免費額度
臨時用 → 筆電 + Tunnel；正式長期 → 上雲（OCI）。同一個 Docker image，兩種都能跑。
10 · 部署方案 C（雲端）

上 Oracle Cloud：固定網址、24 小時

  1. 建立永久免費 VM（Ubuntu）
  2. 開放連接埠（雲端 + VM 兩層）
  3. 安裝 Docker
  4. 上傳程式並 docker compose up
  5. 接網域 + 自動 HTTPS（Caddy）

好處

  • 固定網址：QR 不用每次重出
  • 24 小時在線：不依賴任何人筆電
  • 永久免費額度內，零成本
  • 正式級、可長期當公司工具
已備妥完整 DEPLOY_OCI.md 步驟文件。
11 · 安全與隱私

資料安全怎麼把關

🔑 管理金鑰保護

  • 開關場次、看名單、匯 CSV → 需金鑰
  • 簽到頁公開（大使要能掃）
  • 金鑰錯誤一律擋下（401）

🛡️ 隱私設計

  • 簽到下拉只露姓名，不外洩信箱
  • 完整名單與報表僅管理員可見
  • 資料存自有資料庫，可隨時備份
分享 Docker 版給外部時，會以「範例名單」取代真實名冊，避免內部資料外流。
12 · 結論

三方案總覽與建議

方案適用情境網址需要顧著
A 本機直跑自己、臨時測試臨時、會變電腦開著
B 筆電 Docker + Tunnel臨時活動、給別人快速用臨時、會變電腦開著
C 雲端 Docker（OCI）正式、長期固定無（24h）

建議

近期活動先用 B（筆電 Docker + Tunnel），零成本即可上線；
確定常態使用後，一鍵把同一個 image 搬到 C（OCI 雲端），取得固定網址與 24 小時穩定。
