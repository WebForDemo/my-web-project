Status Bar 耳機 Icon 異常調查報告

現象：Android 16 裝置 Status Bar 的 WiFi icon 左側出現不明 icon，每次開機都存在

平台：MTK Android 16 (MT8171 / MT6366)
裝置：EA0T4A (e-reader)
發現日期：2026-05-28
修正版本：kernel/kernel_device_modules-6.6

1. 問題現象

觀察 說明
異常現象 WiFi icon 左側多出一個耳機形狀的 icon，沒有插耳機也出現，每次開機都存在
ADB 可控制 adb shell am broadcast -a android.intent.action.HEADSET_PLUG --ei state 0 → icon 消失
adb shell am broadcast -a android.intent.action.HEADSET_PLUG --ei state 1 → icon 出現
getevent 確認 adb shell getevent -S 顯示 mt8171-mt6366 Headset Jack（event3）的 switch 狀態為 0004
換算：0x0004 = bit 2 = SW_HEADPHONE_INSERT = 1，Kernel 在無耳機下就回報耳機已插入

▼ adb shell getevent -S 實際輸出（問題裝置）

device 5: mt8171-mt6366 Headset Jack switch state: 0004 ← SW_HEADPHONE_INSERT = 1

2. Icon 識別 — 是耳機 icon，不是衛星

重要釐清 初步推測這是衛星通訊 icon（satellite slot），但 ADB HEADSET_PLUG broadcast 測試證明這是 耳機 icon（headset slot）。

Status Bar icon 排列順序（定義於 frameworks/base/core/res/res/values/config.xml）：

config_statusBarIcons 陣列（節錄）： ... [5] headset ← 耳機 icon（這個！） ... [30] bluetooth [31] ethernet [32] satellite ← 衛星 icon（不是這個） [33] wifi ← WiFi icon ...

headset (位置 5) 與 wifi (位置 33) 之間的所有 icon 因條件未觸發而不顯示，視覺上 headset 就緊靠在 WiFi 左邊。

4. 背景知識：耳機偵測的兩種 Kernel 介面

舊版（Android 早期）

/sys/class/switch/h2w/state

  • 由 android_switch driver 提供
  • WiredAccessoryManager 直接讀 sysfs 節點
  • 這台裝置 沒有 這個路徑
  • cat /sys/class/switch/h2w/state → No such file

新版（這台裝置）

/dev/input/eventX（EV_SW events）

  • 由 ALSA sound driver 建立 input device
  • 透過 snd_soc_jack_new() 建立 "Headset Jack"
  • InputManagerService 讀取 SW_HEADPHONE_INSERT bit
  • getevent -S 可看到當前狀態

getevent -S 輸出解讀（問題裝置）：

add device 5: /dev/input/event3 name: "mt8171-mt6366 Headset Jack" switch states: 0004 0x0004 = binary: 0000 0000 0000 0100 ↑ bit 2 = SW_HEADPHONE_INSERT = 1 ← 問題所在

5. 背景知識：ACCDET 耳機偵測原理

ACCDET（Accessory Detector）是 MTK PMIC（MT6366）內建的耳機偵測電路，透過測量 3.5mm 接頭腳位的阻抗來判斷插入狀態：

PMIC 內部電路示意： MICBIAS1（偏壓） → [R_bias] → EINT pin ──→ 耳機孔 MIC 腳 ↓ Comparator（閾值比較） ↓ GND ──────────── 耳機孔 GND 腳
狀態 EINT pin 電位 ACCDET 判斷 SW_HEADPHONE_INSERT
有耳機孔、未插 Pull-up 到高電位（設計保證） No headset 0
有耳機孔、插入有麥耳機 被耳機拉低（低阻抗） Headset with mic 1 + SW_MICROPHONE_INSERT=1
有耳機孔、插入無麥耳機 中等阻抗 Headphone only 1
沒有耳機孔（這台裝置） 浮接（floating），阻抗不確定 誤判為 headphone 1（誤報）

沒有耳機孔的裝置，EINT pin 懸空，偏壓電流將其拉到一個落在偵測閾值內的電位，被 Comparator 誤判為「耳機插入」。這也是 Kconfig 標注 "Select N if you don't have jack on board" 的原因。

8. 修正方案

修正已完成 共修改兩個檔案：

檔案 1：Kernel Config

檔案：kernel/kernel_device_modules-6.6/kernel/configs/mt8171.config 修改： CONFIG_DEVICE_MODULES_SND_SOC_MT6358=m ← 保留（其他音訊功能需要） + CONFIG_SND_SOC_MT6366_ACCDET=n ← 新增：關閉 ACCDET 效果： - mt6358-accdet.ko 不再被編譯 - mt8171-mt6366.c 中的 accdet_init 呼叫因 IS_ENABLED() guard 被跳過 - "mt8171-mt6366 Headset Jack" input device 不再被建立

檔案 2：ko_order_table（避免 build 找不到 .ko 而失敗）

檔案：device/ntx/EA0T4A_A00/ko_order_table.csv 修改： - mt6358-accdet.ko,/../kernel/kernel_device_modules-6.6/... ← 刪除此行 原因：CONFIG_SND_SOC_MT6366_ACCDET=n 後 .ko 不存在， 保留此行會導致 build 時找不到檔案而報錯。
