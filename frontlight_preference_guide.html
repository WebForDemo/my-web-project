FrontLightPreference 技術講解

E-ink 前光控制元件 ─ MtkSettings 模組
MtkSettings/src/com/android/settings/display/FrontLightPreference.java
1 角色定位

這是一個自訂 Preference 元件，繼承自 SwitchTwoSeekbarPreference（一個開關 + 兩條拉桿的佈局）。
它負責在 Settings → 顯示設定 裡提供 E-ink 前光的三項控制：

  • 開關 ─ 前光 ON / OFF
  • SeekBar 1 ─ 亮度 (0 ~ 255)
  • SeekBar 2 ─ 色溫 (0 ~ 255)
2 資料庫 Keys（唯一真理來源）
Key 用途 範圍
screen_brightness 前光亮度 0 ~ 255
screen_brightness_color 前光色溫 0 ~ 255
screen_front_light_mode 前光開關 0 / 1
3 生命週期
onAttached ─ 註冊 ContentObserver
onBindViewHolder ─ 初始化 UI
onDetached ─ 解除 ContentObserver
// onAttached — 監聽三個 Settings.System 欄位的變化 context.getContentResolver().registerContentObserver( Settings.System.getUriFor("screen_brightness"), ...); Settings.System.getUriFor("screen_brightness_color"), ...); Settings.System.getUriFor("screen_front_light_mode"), ...); // onDetached — 解除監聽，避免記憶體洩漏 context.getContentResolver().unregisterContentObserver(...);

5 跨元件同步機制 — ContentObserver

透過 ContentObserver 監聽 Settings.System 資料庫。
不管是誰修改了資料庫，UI 都會即時自動更新。

6 方法速查表
方法 功能
init() 取得 PowerManager 實例
onBindViewHolder() 綁定 UI 元件、讀取初始狀態、設定監聽器
getBrightnessLevel() 從 DB 讀取亮度值 (0~255)
setBrightnessLevel() 寫 DB + 呼叫 setBacklightBrightness()
getBrightnessColourLevel() 從 DB 讀取色溫值 (0~255)
setBrightnessColourLevel() 寫 DB + 呼叫 setFrontlightBrightnessColor()
setFrontLightMode() 寫 DB + 開/關硬體前光
broadcastEinkMode() 發送 E-ink 更新模式廣播 (DU ⇄ 正常)
checkSettingsSystemKey() 檢查 DB 中某個 key 是否存在
changeTextDrawableEnable() 切換 icon 圖示 (亮/暗)
