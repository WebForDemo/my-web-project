2. Android 12 — NtxSettings 獨立 APK 架構

概述

Android 12 時代的 NtxSettings 是一個完全獨立的 APK（com.android.ntxsettings），它不繼承 AOSP Settings 的任何 code，而是用一個簡單的 Activity 搭配手刻兩欄 Layout 來實現所有設定功能。

  • 左側 Tab 是用 LinearLayout 動態加入的純文字按鈕（含圖示），每個 Tab 高度 72dp、字體 28dp。
  • 右側內容是一個 FrameLayout 搭配 title bar，透過 FragmentTransaction.replace() 切換內容。
  • 全域 Dialog、Radio Button 樣式已在 Frameworks 底層（Theme.DeviceDefault）被改寫為電子紙風格，因此 Settings XML 不需額外指定 Dialog Theme。

3. Android 16 — AOSP 原生 MtkSettings 架構

概述

AOSP 原生的 Settings 完全捨棄了兩欄設計，改用單頁式 Material 3 長列表。首頁是一個帶有 AppBarLayout、搜尋列、大標題、Avatar 頭像的現代化界面。所有子設定頁面都透過 Activity Embedding（分屏嵌入）在大螢幕上展示雙欄。

  • 首頁 Fragment：TopLevelSettings，巨大的 RecyclerView 偏好清單。
  • 子頁面：點擊後啟動新的 SettingsActivity（或透過 AE 嵌入右側面板）。
  • 沒有手動實作 Tab，沒有強制分隔線，沒有手工 Title Bar。
  • Dialog 使用 Material 3 圓角風格。

4. 現行方案 — NtxSettings 注入 MtkSettings

概述

我們把 Android 12 的兩欄式 E-ink UI 直接嫁接進 MtkSettings 的 SettingsHomepageActivity。
利用 SharedPreferences("PREF_NTXSETTINGS") 作為開關：

  • DEFAULT_SETTINGS_ITEMS = true（預設）→ 使用 activity_ntx_settings.xml 兩欄佈局 + Theme.NtxSettings
  • DEFAULT_SETTINGS_ITEMS = false → 走原生 AOSP A16 路徑，不做任何干涉
  • 兩條路徑完全隔離，Ntx 模式不會觸碰 AOSP 的 CoordinatorLayout / AppBarLayout / Activity Embedding

onCreate() 分支決策流程

protected void onCreate(Bundle savedInstanceState) {
    SharedPreferences pref = getSharedPreferences("PREF_NTXSETTINGS", MODE_PRIVATE);
    boolean ntxMode = pref.getBoolean("DEFAULT_SETTINGS_ITEMS", true);

    if (ntxMode) {
        setTheme(R.style.Theme_NtxSettings);    // ← NoActionBar + 大字體 + NtxAlertDialog
    }

    // ... 系統前置檢查 (provisioned, embedding, task root) ...

    if (ntxMode && !isAlreadyEmbedded) {
        setContentView(R.layout.activity_ntx_settings);  // ← 走 A12 兩欄路徑
        initNtxView();      // 綁定 tabContainerLayout, titleTextView, switchBar
        initNtxTab();        // 動態 inflate Tab，綁定 tabClickListener
        return;             // ⚠️ 重要：直接 return，不執行 A16 初始化
    }

    // ===== 以下是 A16 原生路徑 =====
    setupEdgeToEdge();
    setContentView(R.layout.settings_homepage_container);
    // ... TopLevelSettings, AppBarLayout, Suggestions, Avatar ...
}

10. ItemDecoration（分隔線 / Padding）

SettingsPreferenceFragment 中的自訂 ItemDecoration

在 Ntx 模式下（PREF_NTXSETTINGS = true），在 onCreateRecyclerView() 中強制加入自訂 RecyclerView.ItemDecoration：

recyclerView.addItemDecoration(new RecyclerView.ItemDecoration() {
    private Paint paint = new Paint();
    { paint.setColor(Color.BLACK); }

    // 每個 item 的上下左右間距
    void getItemOffsets(Rect outRect, ...) {
        outRect.top    = 20;   // 20px 上間距
        outRect.bottom = 20;   // 20px 下間距
        outRect.left   = 32;   // 32px 左內縮
        outRect.right  = 32;   // 32px 右內縮
    }

    // 強制手繪黑線 divider（帶左右 margin）
    void onDraw(Canvas c, RecyclerView parent, ...) {
        for (int i = 0; i < parent.getChildCount(); i++) {
            View child = parent.getChildAt(i);
            c.drawRect(
                32,                           // left margin
                child.getBottom() + 18,        // y-start
                parent.getWidth() - 32,        // right margin
                child.getBottom() + 20,        // y-end (2px thick)
                paint
            );
        }
    }
});
⚠️ 設計哲學： A16 的 Material 3 刻意隱藏所有分隔線（留白分隔），但電子紙螢幕因對比度低且無法靠色彩深淺暗示分隔，必須強制加回黑線。 透過 setDivider(null); setDividerHeight(0); 停用系統預設 divider，再用自訂 ItemDecoration 畫出帶 margin 的黑線，避免「雙畫」問題。
