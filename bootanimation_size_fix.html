Bootanimation Size Fix

開機動畫尺寸不符合螢幕解析度

這份說明整理本次問題原因、修改內容與驗證方式。目標是讓開機動畫尺寸與裝置實際螢幕解析度一致，避免畫面在開機階段顯示偏小或未滿版。

問題摘要

裝置透過 adb shell wm size 查到的實體螢幕尺寸是 1860x2480，但原始 bootanimation 的設定與圖片 frame 都是 1404x1872。 兩者長寬比相同，都是 3:4，因此不是比例錯誤，而是整套動畫解析度較小。

螢幕實體尺寸 1860 x 2480
原始動畫尺寸 1404 x 1872
修正後動畫尺寸 1860 x 2480
!
根本原因

desc.txt 宣告的動畫畫布尺寸與 PNG frame 實際尺寸都小於螢幕實體解析度，所以系統播放時會依照較小畫布處理，導致開機動畫無法完整貼合螢幕尺寸。

修改方式

  1. 確認裝置螢幕實體尺寸：adb shell wm size，結果為 Physical size: 1860x2480。
  2. 檢查 bootanimation/desc.txt，原本第一行為 1404 1872 2。
  3. 檢查 bootanimation/part0/*.png，所有 frame 原本都是 1404 x 1872。
  4. 將 desc.txt 第一行更新為 1860 2480 2。
  5. 把 part0 內 10 張 PNG 重新縮放為 1860 x 2480，並維持 RGBA 格式。
  6. 使用不壓縮方式重新打包 bootanimation.zip，確保 entry 仍是 Stored 0%。
desc.txt 修改後內容：
1860 2480 2
p 0 0 part0

驗證結果

✓
目前檔案已完成修正

bootanimation.zip 內的 desc.txt 已更新為 1860 2480 2， 10 張 PNG frame 都是 1860 x 2480，且 zip 內容保持 Stored 0% 格式。

建議刷入後再於實機開機流程確認一次，重點觀察動畫是否滿版、是否有黑邊，以及是否有被裁切。
