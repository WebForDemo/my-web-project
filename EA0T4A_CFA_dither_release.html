1Dithering 是什麼？（白話版）

E-Ink 的灰階限制

模式灰階數說明
Y12 色只有純黑、純白
Y416 色黑到白共 16 階
Y8256 色最細緻，刷新最慢

Dithering 要解決的問題

假設圖片有 256 種灰，但螢幕只有 2 色（Y1）。

沒有 Dithering：強制把 256 色壓成 2 色 → 細節丟失，色塊分明、很醜
有 Dithering：把黑白像素排成特定花紋，視覺上欺騙眼睛，感覺有更多層次

視覺效果對比

從深灰漸變到淺灰（只有黑白可用）
❌ 沒有 Dithering
████████░░░░░░░░
（硬切邊界，突兀）
✅ 有 Dithering
████▓▓▒▒░░░░░░
（漸層感，自然）
類比：報紙印刷
靠近看只有黑點，遠看卻有灰色。Dithering 就是同樣的原理。

2模式對照與使用場景

模式灰階速度Dither 參數典型用途
Y1 2 色 最快 flags=0x2000 + dithermode 可選 Seekbar 滑動、手寫輸入、拖曳物件
Y4 16 色 中等 flags=0x4000（dithermode 寫死，無須設定） 需要層次感的動態畫面
GC16 16 色 慢 ~330ms —（不使用 Dither） 高品質靜態畫面
GLR16 / GLC16 16 色 慢 —（不使用 Dither） 文字優化渲染

Y1 的三種 dithermode 演算法

值演算法特性
0x300Quantization（Q）最基本，邊緣硬
0x301Blue Noise（B）初步看起來效果最好 ★
0x302Sierra（S）預設值，漸層均勻

使用原則

  • 理論上都用 Y1，速度最快，搭配 dither 補足層次
  • Y1 dithermode 推薦先測 0x301 Blue Noise
  • Y4 dithermode 寫死，不需設定，直接填 flags=0x4000
  • Global Mode 為主；NNote 類似 Lasso 拖曳，System 是 Seekbar / 滑動

3參數說明

必填參數

參數作用常用值
waveformmode 選波形，決定刷新品質 1 開啟 / 0 關閉（同時關 dither）
flags 選 Dither 類型 0x2000 = Y1 ／ 0x4000 = Y4

選填參數

參數作用常用值
updatemode 刷新範圍 0 = 全螢幕
waitmode 是否等刷新完才繼續 1 = 等待
dithermode Y1 的演算法（Y4 不用填） 0x300 Q / 0x301 Blue Noise / 0x302 Sierra

flags vs dithermode 的關係

flags → 決定「用幾個灰階」做 Dither
（黑白兩色 Y1？還是 16 色 Y4？）
dithermode → 決定「點要怎麼排」
（斜排 Q？隨機 Blue Noise？菱形 Sierra？）
類比印刷
flags = 決定「印幾色」
dithermode = 決定「點要怎麼排」

關閉 Dither

adb shell am broadcast \
  -a ntx.eink_control.GLOBAL_REFRESH \
  --ei waveformmode 0

waveformmode=0 即同時關閉 dither，其他參數不需要填

4快速啟用指令

Y1 Dither（推薦，速度最快）

# Blue Noise（初步看起來效果最好）★
adb shell am broadcast \
  -a ntx.eink_control.GLOBAL_REFRESH \
  --ei updatemode 0 --ei waitmode 1 \
  --ei waveformmode 1 \
  --ei flags 0x2000 --ei dithermode 0x301

# Sierra（預設值）
adb shell am broadcast \
  -a ntx.eink_control.GLOBAL_REFRESH \
  --ei updatemode 0 --ei waitmode 1 \
  --ei waveformmode 1 \
  --ei flags 0x2000 --ei dithermode 0x302

# Quantization
adb shell am broadcast \
  -a ntx.eink_control.GLOBAL_REFRESH \
  --ei updatemode 0 --ei waitmode 1 \
  --ei waveformmode 1 \
  --ei flags 0x2000 --ei dithermode 0x300

Y4 Dither（16 色，較豐富）

# dithermode 寫死，無須設定
adb shell am broadcast \
  -a ntx.eink_control.GLOBAL_REFRESH \
  --ei updatemode 0 --ei waitmode 1 \
  --ei waveformmode 1 \
  --ei flags 0x4000

關閉 Dither

# 任何時候都可以用這個關閉
adb shell am broadcast \
  -a ntx.eink_control.GLOBAL_REFRESH \
  --ei waveformmode 0
dithermode 預設值：不填 dithermode 時預設為 0x302 Sierra

5測試流程

1
基準：不開 Dither 
adb shell am broadcast \
  -a ntx.eink_control.GLOBAL_REFRESH \
  --ei updatemode 0 --ei waitmode 1 \
  --ei waveformmode 0
2
Y1 Dither（三種演算法依序測試） 
# 2-A  Sierra（先測這個）
... --ei flags 0x2000 --ei dithermode 0x302

# 2-B  Blue Noise ★推薦
... --ei flags 0x2000 --ei dithermode 0x301

# 2-C  Quantization
... --ei flags 0x2000 --ei dithermode 0x300
每測完一個，先關閉再換下一個
↓
每次切換前：關閉 Dither 
adb shell am broadcast \
  -a ntx.eink_control.GLOBAL_REFRESH \
  --ei waveformmode 0
3
Y4 Dither（dithermode 寫死） 
adb shell am broadcast \
  -a ntx.eink_control.GLOBAL_REFRESH \
  --ei updatemode 0 --ei waitmode 1 \
  --ei waveformmode 1 --ei flags 0x4000
4
驗收：確認關閉正常 
adb shell am broadcast \
  -a ntx.eink_control.GLOBAL_REFRESH \
  --ei waveformmode 0
測試順序：基準 → 2-A Sierra → 關 → 2-B Blue Noise → 關 → 2-C Q → 關 → Y4 → 關 → 驗收

每個 case 確認清單

🌫️

看得出層次，不斷層、不色塊分明
🔤

清晰，沒有花斑或鋸齒感
✅
關閉後
恢復正常顯示，無殘留花紋

Y1 三種演算法比較

演算法優點缺點
0x302 Sierra 漸層均勻，預設值 較規律，可能產生條紋感
0x301 Blue Noise ★ 隨機分布，自然感最強 —
0x300 Quantization 計算最簡單 邊緣較硬，層次感弱
初步推薦：Blue Noise（0x301）效果最自然，先以此為主進行驗收。最終選擇依實機視覺效果而定。
