nNote android16 升級
工程接手簡報

把 nNote 從舊的 android12 工具鏈升級到 android16，並改用平台 eink.jar（含 EinkManager / HandWriteSuspendController）。本簡報以「乾淨的 nNote 原始碼」為起點，照著每頁做即可重現全部改動。

android12 → android16 Gradle 8.6 · AGP 8.5.0-alpha07 · JDK 21 不含閃爍修正（另案）

用 ← → 或下方按鈕翻頁 · 共 11 頁

OVERVIEW

做什麼 · 為什麼

整體只做兩類事，其餘都是為了讓這兩件成立的相依調整。

① 升工具鏈

android16 的 framework/eink jar 是 Java 21，舊的 Gradle 7.2 / AGP 7.1.3 / JDK 17 吃不下（record、class 65 等）。必須升到 Gradle 8.6 / AGP 8.5 / JDK 21。

② 換 eink

app 自帶的 com.ntx.eink.* source 換成平台整顆 libntx_eink.jar，順帶取得 HandWriteSuspendController。

範圍界定：本簡報只涵蓋「工具鏈 + eink jar 遷移」。e-ink 閃爍修正是獨立議題，不在此。
ARCHITECTURE

架構圖：jar 怎麼進到 App、怎麼跟裝置溝通

一句話：framework 是「編譯期借用、裝置提供」→ compileOnly；eink.jar 是「裝置沒有、要自己帶」→ implementation 打包。

STEP 1 – 2

準備 jar 與 Gradle wrapper

1 · 兩顆 jar 各就各位

jar放哪 / 怎麼用
framework-minus-apex.jarcompileOnly，留在平台輸出原地，路徑寫進 build.gradle
libntx_eink.jar複製到 app/libs/，由 fileTree 自動打包
# eink jar 複製進專案
cp jars/libntx_eink.jar <專案>/app/libs/libntx_eink.jar
別把 framework jar 放進 app/libs（~48MB，會被 fileTree 誤打包）。它只走 compileOnly。

2 · gradle/wrapper/gradle-wrapper.properties

- distributionUrl=…/gradle-7.2-all.zip
+ distributionUrl=https\://services.gradle.org/distributions/gradle-8.6-bin.zip
STEP 3 – 4

build.gradle 與 gradle.properties

3 · 升 AGP/Kotlin、定義 framework 路徑、移除舊 hack

-   ext.kotlin_version = '1.6.10'
+   ext.kotlin_version = '1.9.24'
-   classpath 'com.android.tools.build:gradle:7.1.3'
+   classpath 'com.android.tools.build:gradle:8.5.0-alpha07'
+ ext.ntxFrameworkJar = '/mnt/aosp2/.../framework-minus-apex_intermediates/classes.jar'
  // allprojects 裡的 -Xbootclasspath/p:libs/classes.jar 整段刪掉

4 · gradle.properties

+ org.gradle.java.home=/usr/lib/jvm/java-21-openjdk-amd64
+ android.nonTransitiveRClass=false   # app R 保留 library 資源
+ android.nonFinalResIds=false        # R.id 維持 final（switch 用）
  # android.enableJetifier=true 保留（舊 support-lib 依賴要）
非 ASCII 路徑陷阱：framework 路徑寫在 build.gradle 的 ext，別寫進 gradle.properties（它以 ISO-8859-1 讀，「桌面」之類會亂碼 → 找不到檔）。
STEP 5

app/build.gradle（AGP 8 相容）

- apply plugin: 'kotlin-android-extensions'   # Kotlin 1.8+ 已移除，沒用到
- compileSdkVersion 31
+ compileSdkVersion 34
  // 刪掉 gradle.projectsEvaluated{...bootstrapClasspath...libs/classes.jar} 整段
+ buildFeatures { buildConfig true; aidl true }   # AGP 8 預設關閉
  buildTypes { debug {
-     minifyEnabled true;  shrinkResources true
+     minifyEnabled false; shrinkResources false  # debug 關 R8
  } }
- lintOptions { … }
+ lint       { … }
  dependencies {
-   implementation fileTree(include:['*.jar','*.aar'], dir:'libs')
+   implementation fileTree(include:['*.jar','*.aar'], exclude:['classes.jar'], dir:'libs')
-   compileOnly files('libs/classes.jar')
+   compileOnly files(rootProject.ntxFrameworkJar)
  }
要點：framework = compileOnly（不打包）；libntx_eink.jar 在 app/libs 由 fileTree 自動 implementation 打包。
STEP 6

換 eink：放 jar + 刪 app 自帶 source

整顆 jar 內含 EinkManager / IEinkService / IEinkMode，跟 app 自帶的同名，必須刪掉 app 自帶的，否則 Duplicate class。

cp jars/libntx_eink.jar app/libs/libntx_eink.jar
rm app/src/main/java/com/ntx/eink/EinkManager.java
rm app/src/main/aidl/com/ntx/eink/IEinkService.aidl
rm app/src/main/aidl/com/ntx/eink/IEinkMode.aidl
rm app/libs/classes.jar        # 舊 android12 框架，已不用
這次能整顆用的原因：app 自帶 EinkManager/aidl 先前已照這版平台 libntx_eink 同步，方法簽名對得上 → 刪掉後 NtxView 等呼叫端零修改即可編過。
只刪 eink 的 aidl：ntx.note.aidl.OcrAidl 等其他 aidl 要留，所以步驟 5 的 aidl true 不能省。
STEP 7 – 9

proguard、IDE JDK、編譯驗證

7 · proguard-rules.pro（只影響 release）

+ -keep,allowobfuscation,allowshrinking class com.google.gson.reflect.TypeToken
+ -keep,allowobfuscation,allowshrinking class * extends com.google.gson.reflect.TypeToken

新版 R8 會剝掉匿名 TypeToken 泛型簽名 → release 執行期 Missing type parameter。debug 已關 minify 用不到。

8 · Android Studio Gradle JDK

一定要設 21：Settings → Build Tools → Gradle → Gradle JDK 選 21。IDE 會蓋過 gradle.properties，不設會報 wrong version 65.0。

9 · 編譯驗證

./gradlew --stop
./gradlew :app:assembleDebug   # → BUILD SUCCESSFUL
  • checkDebugDuplicateClasses 通過 = eink 類別只有一份
  • 產物 app/build/outputs/apk/debug/app-arm64-v8a-debug.apk

DELIVERY & MAINTENANCE

交付物與日後維護

本包交付物

  • nNote_android16_jar升級_接手指南.html（本檔）
  • jars/framework-minus-apex.jar — compileOnly
  • jars/libntx_eink.jar — implementation（含 EinkManager / HandWriteSuspendController）

日後換平台版只要兩步

  • 更新 root build.gradle 的 ext.ntxFrameworkJar 路徑
  • 重新複製新的 libntx_eink_intermediates/classes.jar → app/libs/libntx_eink.jar
若平台版改了方法簽名：刪 app source 後 build 可能出現 cannot find symbol，逐一修 NtxView 等呼叫端對齊即可。
接手者三個要改的絕對路徑：build.gradle 的 ntxFrameworkJar、gradle.properties 的 org.gradle.java.home、app/build.gradle 的簽章 storeFile。
