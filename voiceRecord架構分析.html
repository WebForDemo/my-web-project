一、專案目標

🎙️
一鍵錄音
點擊即錄，即時音波視覺化與計時器
🔒
背景不中斷
Foreground Service 確保螢幕關閉時持續錄音
🔔
通知列控制
暫停 / 停止按鈕，狀態與 App UI 即時同步
📡
EMI 測試
錄音、播放、相機預覽三路並行壓力測試

五、完整功能矩陣

5.1 主頁面功能

功能說明實作方式
📋 錄音列表RecyclerView 按日期降序顯示RecordingAdapter + DiffUtil + LiveData
🏷️ 自動命名「新錄音」→「新錄音 1」→「新錄音 2」Repository.generateDefaultTitle()
✏️ 重新命名展開後點擊標題彈出 AlertDialogMainActivity.showRenameDialog()
🔴 一鍵錄音底部紅色 72dp 圓形按鈕ImageView + bg_record_button.xml

5.2 錄音中介面

功能說明實作方式
⏱️ 動態計時器格式 00:00.00，每 50ms 更新RecordingService → LiveData → TextView
🌊 即時音波紅色線條右→左捲動，20fpsWaveformView 自訂 Canvas View
⏹️ 停止按鈕灰色圓環 + 紅色方形Drawable shape + View click
🎬 滑入動畫面板由底部滑入，300msTranslateAnimation

5.3 播放控制

功能說明實作方式
📂 展開播放器點擊列表項目展開控制面板RecordingAdapter visibility toggle
🎚️ SeekBar可拖動，顯示當前/剩餘時間SeekBar.OnSeekBarChangeListener
▶️⏸️ 播放/暫停圖示即時切換PlaybackState LiveData
⏩ 快進 15sseekTo(current + 15000)RecordingService.seekForward15()
⏪ 快退 15sseekTo(current - 15000)RecordingService.seekBackward15()
🗑️ 刪除確認對話框 → 刪除檔案+記錄RecordingRepository.delete()

5.4 背景執行與通知

功能說明實作方式
🔒 Foreground Service錄音/播放不被系統回收startForeground() + FOREGROUND_SERVICE 權限
🔔 常駐通知顯示檔名 + 時長，即時更新NotificationCompat.Builder
🎮 通知按鈕「暫停/繼續」+「停止」PendingIntent → Service Action
🎧 MediaSession耳機、藍牙按鍵整合MediaSessionCompat + Callback
🔄 UI 同步通知操作即時反映 App 內 UILiveData 雙向觀察

八、MVVM 架構設計優勢

🧩 關注點分離

UI、業務邏輯、資料存取完全解耦，各層獨立修改不互相影響

🧪 可測試性

ViewModel 與 Repository 可獨立進行單元測試，無需 Android 框架

🛡️ 生命週期安全

LiveData 自動感知 Activity 生命週期，避免記憶體洩漏與 Crash

🔌 擴展性

後續功能（雲端同步、分享）在 Repository 層擴展，不影響 UI 層

🏭 MTK 整合

模組化設計便於移植至 MTK 系統開發環境，可獨立替換任一層

🔄 即時同步

LiveData 驅動 UI 自動更新，無需手動 callback，資料永遠一致
