開機圖(BootLogo)
是怎麼顯示的,以及如何置換

從按下電源鍵到進入桌面,螢幕上的每一張圖由誰、在哪個階段畫出來;以及要換成自家圖時,該動哪裡、要符合什麼格式。

EA0T4A_A00 · 電子紙 reader logo 分區(A/B)· BMP 來源 · LK + Kernel 階段
01 / 08 先建立觀念

開機畫面其實是兩套不同系統接力

最容易混淆的一點:「開機圖」和「開機動畫」是兩個完全獨立的階段、不同檔案、不同機制。改錯地方就不會生效。

階段一 · ANDROID 之前

BootLogo（本報告主題）

由 Bootloader(LK)與 Kernel 直接把圖畫到螢幕。圖是 BMP,打包進 logo 分區。在還沒有作業系統時就能顯示。

階段二 · ANDROID 之後

BootAnimation（另一套）

Android 起來後由 SurfaceFlinger 播放 bootanimation.zip(一疊 PNG 循環)。位於 system/media/,屬 userspace。

一句話　BootLogo = 開機「最早」那幾張靜態圖(bootloader/kernel/充電/關機);BootAnimation = kernel 之後的循環動畫。本報告只談前者。

桌面
 

kernel.bmp 是進 kernel 後的第一張圖,會靜止停留整個 kernel + init 載入期間,直到 Android 的 bootanimation 接手——所以它是使用者實際上盯著最久的一張。

分支　關機狀態插充電線 → 不開 Android,進入 KPOC 充電模式,顯示電池/充電 UI;關機時顯示 power_off.bmp。
03 / 08 圖檔對照

哪張圖、什麼時候出現

*_uboot.bmpLK(bootloader)階段的圖尚未打通
*_kernel.bmp進 Kernel 第一張,停留到 Android 接手(最主要)一定換
*_power_off.bmp關機畫面一定換
*_bat_* / battery充電模式:電池外框 + 灌電動畫 + 電量分段全換才動
*_fast_charging_*偵測到快充時的專屬動畫 + 百分比全換才動
*_low_battery*電量過低警告 / 無法開機全換才動
*_num_0~9 / percent拼出電量百分比數字(85% 等)全換才動
*_charger_ov.bmp充電器過壓警告全換才動

前綴 xga2480_ 對應本機面板。最少換 kernel + power_off;要徹底去除 MTK 預設圖則全換。

04 / 08 Build 原理

從 BMP 到 logo 分區 的資料流

原始 BMP 不會直接燒進去。build 會經過「轉 raw → 壓縮 → 打包」三步,產生 logo.img 燒進 logo 分區。

xga2480/*.bmp原始素材(你改這裡)
→
bmp_to_raw_hwtconBMP → raw(電子紙 T-CON)
→
zpipe (zlib)壓縮
→
logo.bin / logo.imgboot_logo + boot_charging
→
logo_a / logo_b燒進分區(A/B)

boot_logo.raw

uboot、kernel 等開機 logo,供 LK / Kernel 依索引讀取顯示。

boot_charging.raw

電池、充電、低電量、數字等充電模式 UI 圖,供 KPOC 充電流程使用。

關鍵　你只要丟「標準 BMP」進素材夾,轉 raw / 壓縮 / 打包都由 build rule 自動完成,不用自己做特規轉換。
05 / 08 解析度 Profile

為什麼是 xga2480,不是別的

BootLogo 下有數十套 xgaXXXX / fhd / wqhd … 資料夾,每套對應一種面板解析度。build 只會挑一套——由產品設定決定。

# device/ntx/EA0T4A_A00/ProjectConfig.mk BOOT_LOGO = xga2480 # ← 這行決定挑哪套素材夾 # logo/rules.mk:未設定時的預設 BOOT_LOGO ?= fhdplus2300

本機正解

xga2480
= 1860 × 2480(對上面板)

常見誤改

xga1680
= 1264 × 1680(別的面板,改了沒用)

判斷依據

看 ProjectConfig.mk 的 BOOT_LOGO,不要用猜的

踩雷點　改錯 profile(例如改 xga1680)build 根本不會用到,開機圖不會變。動手前先確認 BOOT_LOGO 的值。
06 / 08 實作步驟

如何把圖換成自家的

1備好圖,轉成合格 BMPkernel / power_off:1860×2480、24-bit BMP(BMP3、無壓縮、無 alpha)。PNG 要壓平透明。
2放進正確素材夾,沿用原檔名覆蓋 …/BootLogo/logo/xga2480/xga2480_kernel.bmp 等。檔名不能改(build rule 寫死)。
3重新 build logo產生新的 logo.img(build 自動跑 bmp_to_raw → zpipe → 打包)。
4燒 logo 分區並驗證flash logo.img 到 logo_a/b,重開機看 kernel 圖是否換成自家圖。
# PNG → 合格 BMP（ImageMagick） convert in.png -resize 1860x2480! -background black -flatten -type TrueColor \ BMP3:.../BootLogo/logo/xga2480/xga2480_kernel.bmp
07 / 08 限制與注意

容易出錯的五件事

  • profile 要對 — 本機是 xga2480;改到 xga1680 等於沒改。依 ProjectConfig.mk 確認。
  • 檔名不能改 — build rule 依固定檔名抓圖,改名會抓不到。只換內容,不動名稱。
  • 格式要符合 — kernel/power_off = 1860×2480 / 24-bit BMP;PNG 的 alpha 要先壓平。
  • uboot 尚未打通 — *_uboot.bmp 是 LK 階段,目前改了不會顯示(且尺寸/位深不同),先擱置。
  • logo 是獨立分區 — 換圖要燒 logo 分區,不是 system;OTA / fastboot 都要包含它才會更新。
