01功能總覽

此功能為 Android E-Ink 裝置新增 SMode（Special Mode） 刷新控制機制，允許上層 App 透過 Broadcast Intent 動態切換 E-Ink 面板的刷新模式。

🌐 Global Mode

全域刷新模式，持續生效直到關閉（waveformmode=0）。每次 atomicPostBuffer() 都會套用。

ntx.eink_control.GLOBAL_REFRESH

⚡ Quick Mode

單次快速刷新，使用後自動清除。會觸發 scheduleComposite() 強制合成一次。

ntx.eink_control.QUICK_REFRESH

⏳ Wait Mode

控制 DRM WAIT_COMPLETE 屬性，等待 E-Ink panel 刷新完成才返回（同步刷新）。

嵌入 Global / Quick 參數中

🎯 Special Mode

帶額外參數（count / time / region）的特殊刷新模式。

ntx.eink_control.SPECIAL_REFRESH

Broadcast 參數說明

參數說明值域
updatemodefull update (1) / partial update (0)0 or 1
waitmode等待刷新完成 Enable (1) / Disable (0)0 or 1
waveformmode波形模式 (du, gc4, gc8, gc16 等)整數值

03值編碼格式

模式資訊被編碼為 32-bit 整數，透過 getDisplayAttribute() 的 attribute 參數傳遞到 HAL。

32-bit Attribute 編碼

0xA2~A7
Bit 31~24: NTX Mask
U
Bit 23: Update
W
Bit 22: Wait
—
Bit 21~16: Reserved
waveformmode
Bit 15~0: Waveform

NTX Mask 定義

Mask值用途
NTX_SLEEP_MODE_MASK0xA0000000Sleep 模式
NTX_DROP_MODE_MASK0xA1000000丟幀模式
NTX_GLOBAL_MODE_MASK0xA2000000Global 刷新模式
NTX_QUICK_MODE_MASK0xA3000000Quick 刷新模式
NTX_SPECIAL_MODE_MASK0xA4000000Special 模式
NTX_SPECIAL_COUNT_MASK0xA5000000Special 計數
NTX_SPECIAL_TIME_MASK0xA6000000Special 時間
NTX_SPECIAL_REGION_MASK0xA7000000Special 區域

07設計要點與注意事項

⚠️
getDisplayAttribute() 被挪用為寫入通道
此 API 本意是唯讀查詢 display 屬性，但此處被用來傳遞控制指令到 HAL 層。outvalue 返回值被忽略。這是非正規的 HAL 通訊手法，但在 MTK E-Ink 平台上是常見做法，因為標準 HWC HAL 沒有提供自訂命令介面。
💡
Quick Mode vs Global Mode 差異
Quick Mode 是一次性的：套用後 setSModeInfo(0) 立即清除，且會觸發 sfdo_scheduleComposite() 強制合成一次。
Global Mode 是持續性的：一旦開啟，每次 atomicPostBuffer() 都套用設定的 waveform，直到收到 waveformmode=0 關閉。
⏳
Wait Mode 的作用
控制 DRM plane property DRM_PROP_PLANE_WAIT_COMPLETE。設為 1 時，DRM atomic commit 會等待 E-Ink panel 完成刷新才返回（同步刷新），適用於需要確保畫面更新完成後再進行下一步操作的場景。
📝
Phase 2 中被停用的功能
ntx.eink_control.DEFAULT_MODE action 被移除。DropFrames 的 mSurface.updateAuxRefreshMode() 呼叫被註解掉，表示 Drop mode 目前暫未啟用。

08測試指令

# Quick mode — 全螢幕刷新，等待完成，使用 waveform 2 (GC16)
am broadcast -a ntx.eink_control.QUICK_REFRESH \
  --ei updatemode 1 --ei waitmode 1 --ei waveformmode 2

# Global mode — 開啟（partial update，等待完成，使用 waveform 1 = DU）
am broadcast -a ntx.eink_control.GLOBAL_REFRESH \
  --ei updatemode 0 --ei waitmode 1 --ei waveformmode 1

# Global mode — 關閉
am broadcast -a ntx.eink_control.GLOBAL_REFRESH \
  --ei waveformmode 0
