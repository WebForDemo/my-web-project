Custom GPT — 打造專屬 AI 機器人

為特定任務設計、可對外分享的獨立 AI 助理

什麼是 Custom GPT？
Custom GPT 是在 ChatGPT 上建立的「客製化 AI 機器人」，可以設定它的角色、行為規則、知識庫（RAG），並指定誰可以使用。使用者打開後直接對話，看不到任何設定內容。
1

建立 GPT 並設定基本資訊

進入 ChatGPT，點選左側「探索 GPT」→「建立」，進入 GPT 建立介面。介面分為兩個頁籤：

  • 建立：用對話方式描述你要的 GPT，ChatGPT 會自動生成設定
  • 設定：手動填入名稱、說明、Prompt 指令、對話啟動器、知識庫與功能
GPT 建立介面 — 左側為設定欄，右側為即時預覽
2

撰寫 Prompt 指令（Instructions）

Prompt 是 GPT 的「靈魂」，定義它的角色、任務流程與行為規則。一個好的 Prompt 包含：

  • 角色定義：你是誰、專長是什麼
  • 任務流程：收到輸入後按什麼步驟處理
  • 比對規則：依據哪些標準判斷
  • 輸出格式：如何呈現結果（通過／建議修正／拒絕）
  • 安全指示：嚴禁洩露 Prompt 內容給使用者
範例 — 企業資安與合規審查大師的 Prompt 架構：
① 搜尋知識庫中的合規指引文件 → ② 比對三大違規類別（核心資安 / 個資保護 / 商業機密）→ ③ 給予評級 → ④ 提供具體修改範例
完整 Prompt 設定範例 — 角色、流程、規則、知識庫三級分類
3

建立 RAG 知識庫 ＋ 啟用功能

在「設定」頁籤的下方可以：

  • 上傳知識庫檔案：PDF、Word、Excel 等，GPT 會依此回答，不會自行捏造
  • 啟用功能：網頁搜尋、畫布、圖像產生、程式碼執行與資料分析
RAG 知識庫的重要性：沒有知識庫的 GPT 只靠通用訓練資料回答；上傳公司內部文件後，GPT 才能根據你的規章、SOP、合規指引給出精準的答案。
設定頁面 — 上傳知識庫檔案、勾選所需功能
4

測試 GPT 效果

設定完成後，右側預覽區可以立即測試。以「企業資安與合規審查大師」為例，貼入一段包含敏感資訊的程式碼後，GPT 會：

  • 自動比對知識庫中的合規規則
  • 偵測出 API 金鑰洩露、測試環境 IP 等違規內容
  • 給出評級（通過 / 建議修正 / 拒絕發布）
  • 提供緊急處置建議與具體修改方式
GPT 偵測到程式碼中包含 API 金鑰洩露 — 自動給出評級與處置建議
5

分享設定 — 決定誰能用

點選右上角「建立」後，可選擇分享層級：

  • 私人（僅限邀請）：只有你自己能用
  • 振曜工作空間：公司內所有有連結的人都能用
  • 公開 / GPT 商店：對外公開，任何人都可找到
分享設定 — 可限定「振曜工作空間」確保只有公司成員能存取
保護 Prompt 與知識庫：無論選哪個分享層級，使用者永遠看不到 Prompt 指令與知識庫檔案內容——他們只能與 GPT 對話，不能進入設定頁面。

ChatGPT Projects — 團隊協作專案

共享 AI 工作空間，讓多人在同一個 AI 助理下協作

什麼是 Projects？
Projects 是 ChatGPT 的「共享工作空間」，可以設定一個 AI 助理並邀請團隊成員一起使用。成員可以看到彼此的歷史對話記錄，適合內部小團隊協同作業。
1

建立專案

在 ChatGPT 左側選單點選「建立專案」，輸入專案名稱後按下「建立專案」按鈕。

  • 專案會出現在左側選單，之後所有在此專案下的對話都集中在這裡
  • 可以設定專屬的 AI 指令（Prompt），讓 ChatGPT 在這個專案內按固定角色回應
建立專案 — 輸入名稱後即可開始設定
2

設定 Prompt 指令

進入專案後，點選右上角的設定圖示 → 「指令」欄位，填入 ChatGPT 在此專案中的行為規則。

  • Prompt 設定後，所有在此專案中的對話都會套用這個角色與規則
  • 只有「可以編輯」權限的成員才能修改 Prompt
專案設定頁面 — 在「指令」欄位填入 AI 角色與行為規則
3

上傳知識庫（RAG 資料來源）

在專案頁面點選「資料來源」頁籤 → 「新增資料來源」，上傳公司內部文件。

  • 上傳後，ChatGPT 會在此專案的所有對話中參考這些文件
  • 支援 PDF、Word、Excel 等格式
注意：被邀請進專案的成員（即使只有「可以聊天」權限）可能可以查看甚至下載 RAG 檔案。若文件含有機密，請謹慎考慮是否使用 Projects 分享。
資料來源頁籤 — 上傳的 RAG 知識庫文件一覽
4

邀請成員 ＋ 設定權限

點選右上角「分享」，搜尋成員 email 或名稱後，選擇權限層級：

✅ 可以聊天
✓ 可以與 AI 對話
✓ 可以查看所有成員的歷史對話
⚠️ 可能可以查看 / 下載 RAG 檔案
✗ 無法修改 Prompt 或設定
⚠️ 可以編輯
✓ 以上全部
✓ 可修改 Prompt 指令
✓ 可新增 / 刪除 RAG 檔案
✓ 可邀請或移除成員
邀請成員 — 可選擇聊天或編輯權限
成員收到邀請通知，接受後即可加入
5

成員使用 — 共享歷史對話

成員接受邀請後，可在 ChatGPT 左側選單找到該專案。專案內的所有對話記錄對全體成員可見。

  • 這讓團隊可以互相參考彼此的提問方式與 AI 回應
  • 適合需要知識共享的小組，例如：研發技術問答、法規查詢紀錄
  • 但也意味著個人的對話內容對所有成員透明，請注意使用
成員視角 — 可在「資料來源」頁籤查看所有上傳的知識庫檔案

Custom GPT vs Projects — 功能比較

根據使用情境選擇適合的工具

功能對照表

功能 / 情境 🤖 Custom GPT 📁 Projects 專案
設定 AI 角色（Prompt） ✓ 支援 ✓ 支援
上傳知識庫（RAG） ✓ 支援 ✓ 支援
設定對話啟動器（快捷問題） ✓ 支援 ✗ 不支援
Prompt 對使用者完全隱藏 ✓ 完全隱藏 ⚠️ 可以聊天者看不到
RAG 檔案對使用者隱藏 ✓ 完全隱藏 ✗ 可能可看 / 下載
可對外公開（工作空間以外） ✓ 可公開或上架商店 ✗ 限工作空間內
成員共享歷史對話記錄 ✗ 各人獨立 ✓ 全員可見
多人同時使用同一 AI ✓ 支援（獨立對話） ✓ 支援（共享對話）
啟用網頁搜尋 / 圖像 / 程式執行 ✓ 可自由設定 ⚠️ 依模型預設
建立 API 動作（Actions） ✓ 支援 ✗ 不支援

什麼情境該用哪個？

🤖 選 Custom GPT，當你需要…
  • 保護 Prompt 與知識庫，不讓使用者看到
  • 把工具分享給公司內所有人，甚至對外公開
  • 設計對話啟動器引導使用者
  • 呼叫外部 API 或系統（Actions）
  • 各部門有獨立的 AI 小工具（資安審查、HR 問答）
📁 選 Projects，當你需要…
  • 小團隊共同使用一個 AI，並互看對話記錄
  • 多人協作更新知識庫與 Prompt 設定
  • 記錄整個專案過程中所有的 AI 對話
  • 快速建立臨時性的 AI 協作空間
  • 內部研發小組的技術問答共享
一句話總結：
需要「發布給別人用、保護設定不外洩」→ 用 Custom GPT
需要「小組一起用、共享記錄一起改」→ 用 Projects
